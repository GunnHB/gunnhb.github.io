## 리눅스 관리자를 위한 명령어
### 프로그램 설치를 위한 dpkg
최신 우분투에서는 패키지(프로그램)을 설치하기 위해 apt 명령을 사용한다. 이는 **dpkg<sup>Debian Package</sup>**를 포함한 확장 개념에 가깝다.

초창기 리눅스는 새로운 프로그램을 설치하는 것이 어려웠다. 이 점을 개선하여 데비안 리눅스에서 윈도우의 `setup.exe`와 비슷하게 설치 후 바로 실행할 수 있는 설치 파일을 제작했는데, 이것이 패키지<sup>Package</sup>이며, 확장자명은 `*.deb`이다.

자주 사용하는 명령어 옵션은 다음과 같다.

- 설치
  - `-i` 또는 `-install`

```bash
dpkg -i 패키지파일이름.deb
```

- 삭제
  - `-r` 또는 `--remove`

```bash
dpkg -r 패키지이름
```

- 패키지 조회

```bash
dpkg -l 패키지이름    # 설치된 패키지에 대한 정보를 보여줌
dpkg -L 패키지이름    # 패키지가 설치한 파일 목록을 보여줌
```

- deb 파일 내용 조회

```bash
pkg -info 패키지파일이름.deb  # 패키지 파일에 대한 정보를 보여줌
```

프로그램의 설치가 획기적으로 편해졌지만 단점으로는 **'의존성'** 문제가 있다. 예를 들어 Firefox는 당연히 X 윈도우 상에서 가동된다. 그런데 만약 X 윈도우가 설치되지 않은 상태에서 Firefox를 설치한다면 의존성으로 인해 설치되지 않을 것이다. 이를 해결한 것이 apt 명령이다.

### 편리한 패키지 설치를 위한 apt
apt 명령은 특정 패키지를 설치하고자 할 때 의존성이 있는 다른 패키지를 자동으로 먼저 설치해준다. 이는 **인터넷을 통해** deb 파일 저장소<sup>Repository</sup>에서 설치할 deb 파일은 물론, 해당 파일과 의존성이 있는 다른 deb 패키지까지 알아서 다운로드한 후 자동으로 설치해준다.

- 기본 설치 방법

```bash
apt -y install 패키지이름
```

`-y` 옵션은 패키지 설치 중 yes/no를 묻는 부분에서 무조건 yes를 입력한 것으로 간주하고 자동으로 넘긴다.

- 패키지 목록 업데이트

```bash
apt update
```

`/etc/apt/sources.list` 파일의 내용이 수정되면 이 명령으로 다운로드 받을 패키지 목록을 업데이트해야 한다.

- 삭제

```bash
apt remove 패키지이름     # 기존 설치된 패키지를 제거
apt purge 패키지이름      # 기존 설치된 패키지를 설정 파일을 포함해 완전히 제거
apt autoremove          # 사용하지 않는 패키지를 모두 제거
```

- 내려받은 파일 제거

```bash
# 설치할 때 내려받기한 파일 및 과거의 파일을 제거
apt clean
apt autoclean
```

또한 `apt-cache`를 통해 패키지를 설치하기 전에 패키지에 대한 정보나 의존성 문제를 미리 확인해 볼 수 있다.

- 패키지 정보 확인

```bash
apt-cache show 패키지이름   # 패키지의 정보를 화면에 출력
```

- 패키지 의존성 확인

```bash
apt-cache depends 패키지이름  # 패키지에 대한 의존성 정보를 출력
```

- 패키지 역의존성 확인

```bash
apt-cache rdepends 패키지이름   # 이 패키지에 의존하는 다른 패키지의 목록을 출력
```

apt 명령과 관련된 설정 파일은 `/etc/apt/sources.list.d` 디렉토리에 있다. 그 중 `ubuntu.sources` 파일이 중요한데, 이 파일에는 apt 명령을 실행했을 때 인터넷에서 ***해당 패키지 파일을 검색하는 네트워크의 주소***가 들어있기 때문이다.

`apt install 패키지이름` 명령을 입력하면 다음과 같이 동작한다.

1. apt install 입력
2. `/etc/apt/sources.list.d/ubuntu.sources` 파일을 열어 URL 주소 확인
3. '우분투 패키지 저장소'에 설치와 관련된 패키지 목록을 요청
4. 전체 패키지 목록 파일만 다운로드
5. 설치할 패키지와 관련된 패키지의 이름을 화면에 출력
6. y 입력 시 패키지 저장소에 설치에 필요한 패키지 파일 요청
7. 설치할 패키지 파일을 다운로드해서 자동 설치

> 3, 4번 과정에서 실제 패키지 파일을 다운로드하는 것이 아닌 패키지의 이름만 가져온다.
{: .prompt-tip}

여기서 '우분투 패키지 저장소'에는 다음 네 가지가 있다.

- main
  - 우분투에서 공식적으로 지원하는 무료 소프트웨어
- restricted
  - 우분투에서 공식적으로 지원하는 유로 소프트웨어
- universe
  - 우분투에서 지원하지 않는 무료 소프트웨어
- multiverse
  - 우분투에서 지원하지 않는 유료 소프트웨어

### 파일 압축과 묶기
리눅스를 사용할 때 자주 마주치는 압축 파일의 확장명은 `xz`, `bz2`, `gz`, `zip`, `Z` 등이 있다.

> 최근에는 압축률이 더 좋은 `xz`나 `bz2`를 주로 사용하는 추세이다.
{: .prompt-tip}

- `xz`
  - 확장명 `xz`로 압축하거나 풀어준다.
  - 압축률이 뛰어나다.

```bash
xz 파일이름         # '파일이름'을 압축파일인 '파일이름.xz'로 만듦 (기존 파일은 삭제)
xz -d 파일이름.xz   # 압축을 풀어 줌 ('파일이름.xz'의 압축을 풀어 '파일이름'으로 만듦)
xz -l 파일이름.xz   # '파일이름.xz' 압축 파일에 포함된 목록과 압축률 등을 출력
xz -k 파일이름      # 압축 후 기존 파일을 삭제하지 않고 그대로 둠
```

- `bzip2`
  - 확장명 `bz2`로 압축하거나 풀어준다.

```bash
bzip2 파일이름          # '파일이름'을 압축 파일인 '파일이름.bz2'로 만듦
# 압축을 풀어 줌 ('파일이름.bz2'의 압축을 풀어서 '파일이름'으로 만듦)
bzip2 -d 파일이름.bz2
bunzip2 파일이름.bz2
bzip2 -k 파일이름       # 압축 후 기존 파일을 삭제하지 않고 그대로 둠
```

- `gzip`
  - 확장명 `gz`로 압축하거나 풀어준다.

```bash
gzip 파일이름           # '파일이름'을 압축 파일인 '파일명.gz'로 만듦
# 압축을 풀어 줌 ('파일이름.gz'의 압축을 풀어서 '파일이름'으로 만듦)
gzip -d 파일이름.gz
gunzip 파일이름.gz
```
- `zip`/`unzip`
  - Windows용과 호환되는 확장명 `zip`으로 압축하거나 풀어준다.

```bash
zip 새로생성될파일이름.zip 압축할파일이름   # '압축할파일이름'을 '새로생성될파일이름.zip'으로 만듦 (기존 파일은 그대로 남아 있음)
unzip 압축파일이름.zip                   # '압축할파일이름.zip'의 압축을 풀어 줌 (압축 파일은 그대로 남아 있음)
```

Windows용 압축 프로그램의 경우 여러 파일을 압축했을 때 하나의 압축 파일이 만들어지지만, 리눅스에서는 '파일 압축'과 '파일 묶기'를 원칙적으로는 별개의 프로그램으로 실행한다. 물론 편의 상 한 번에 할 수 있는 옵션도 존재하는데, 이것이 `tar`이다.

- `tar`
  - 확장명 `tar`로 묶음 파일을 만들거나 묶음을 풀어준다.
  - 동작
    - `c`: 새로운 묶음을 만듦
    - `x`: 묶인 파일을 풀어줌
    - `t`: 묶음을 풀기 전에 묶인 경로를 보여줌
    - `C`: 묶음을 풀 때 지정된 디렉토리에 압축을 풀어줌 (지정하지 않으면 묶을 때와 동일한 디렉토리에 묶음이 풀림)
  - 옵션
    - `f`: 묶음 파일 이름 지정(필수)
    - `v`: 파일을 묶거나 푸는 과정을 보여줌
    - `J`: `tar` + `xz`
    - `z`: `tar` + `gzip`
    - `j`: `tar` + `bzip2`

```bash
tar cvf my.tar /etc/systemd/        # 묶기
tar cvfJ my.tar.xz /etc/systemd/    # 묶기 + xz 압축
tar cvfz my.tar.gz /etc/systemd/    # 묶기 + gzip 압축
tar cvfj my.tar.bz2 /etc/systemd/   # 묶기 + bzip2 압축
tar tvf my.tar                      # 파일 확인
tar xvf my.tar                      # tar 풀기
tar Cxvf newidr my.tar              # newdir에 tar 풀기 (기존에 newdir이 있어야 함)
tar xfJ my.tar.xz                   # xz 압축 해제 + tar 풀기
tar xfz my.tar.gz                   # gzip 압축 해제 + tar 풀기
tar xfj my.tar.bz2                  # bzip2 압축 해제 + tar 풀기
```

### 파일 위치 검색
리눅스에서 특정 파일의 위치를 검색하는 명령어는 다음과 같다.

- `find`
  - 가장 많이 사용되는 명령어
  - 옵션
    - `-name`: 파일명
    - `-user`: 소유자
    - `-newer`: 전, 후
    - `-perm`: 허가권
    - `-size`: 크기
  - action
    - `-print`: 기본 값
    - `-exec`: 외부 명령 실행

```bash
find /etc -name "*.conf"                # /etc 디렉토리 하위에서 확장명이 *.conf인 파일 검색
find /home -user ubuntu                 # /home 디렉토리 하위에서 소유자가 ubuntu인 파일 검색
find ~ -perm 644                        # 현재 사용자의 홈디렉토리 하위에서 허가권 644인 파일 검색
find /usr/bin -size +10k -size -100k    # /usr/bin 디렉토리 하위에서 파일 크기가 10KB~100KB인 파일 검색

# 고급 사용
find ~ -size 0k -exec ls -l {} \;          # 현재 사용자의 홈 디렉토리 하위에서 파일 크기가 0인 파일의 목록을 상세히 출력
find /home -name "*.swp" -exec rm {} \;    # /home 디렉토리 하위에서 확장명이 *.swp인 파일 삭제
```

> `-exec`와 `\;`는 외부 명령어의 시작과 끝을 나타내며, `find` 명령이 `{}`안에서 실행된다고 보면 된다.
{: .prompt-tip}

- `which 실행파일이름`
  - 절대 경로를 포함한 위치 검색

- `whereis 실행파일이름`
  - 실행 파일 및 소스, man 페이지 파일까지 검색

- `locate 파일이름`
  - 파일 목록 데이터베이스에서 검색
  - `find`보다 빠르지만 데이터베이스 업데이트 전 파일은 검색이 불가능

### cron과 at
