## Network Load Balaner
**Network Load Balaner**<sup>NLB</sup>는 OSI 4계층(전송 계층)에서 작동하는 ***TCP, UDP, TLS 트래픽을 처리하는 데 특화된 고성능 로드 밸런서***이다.

### 전송 계층 기반 라우팅 및 초고성능 처리
- L4 라우팅
  - 패킷의 출발지 IP/포트, 목적지 IP/포트, 프로토콜(TCP, UDP) 정보만 확인하여 트래픽을 라우팅한다.

- 초고성능 및 초저지연
  - 패킷의 내용을 깊게 분석하지 않기 때문에 오버헤드가 적고 처리 속도가 매우 빠르다.

- 장기 연결 지원
  - 수개월에서 수년까지 유지되어야 하는 연결을 지원한다.
  - IoT 디바이스 통신, 실시간 멀티플레이 게임 서버, 금융 거래 시스템 등에 필수적이다.

### 고정 IP 및 클라이언트 IP 보존
NLB는 AZ당 하나의 고정 IP 주소를 가진다. (필요한 경우 직접 생성한 탄력적 IP를 할당) 보안 그룹이나 DNS 설정, 서드 파티 파트너사 연동시 ***반드시 고정된 IP 주소를 대상으로 지정해야 하는 환경***에서 필적으로 사용된다.

트래픽을 대상 서버로 전달할 때 패킷의 출발지 IP를 로드 밸런서의 IP로 교체하지 않고 ***클라이언트의 원래 IP 주소를 그대로 유지***한 채로 백엔드 서버에 전달한다. 이를 통해 서버의 OS 단이나 애플리케이션 측에서 접속자의 실제 IP를 바로 식별하고 로깅할 수 있다.

### NLB의 대상 그룹<sup>Target Group</sup>
- EC2 인스턴스
  - TCP/UDP 포트를 열어두고 서비스하는 인스턴스로 트래픽을 분산한다.

- IP 주소
  - VPC 내부의 리소스뿐만 아니라 온프레미스 서버 환경이나 타 클라우드의 서버로 연결할 수 있다.

- ALB
  - NLB의 뒷단에 ALB를 대상 그룹으로 연결하는 아키텍쳐를 구성할 수 있다.
  - 고정 IP가 필요한 요구사항(NLB)과 HTTP/HTTPS 콘텐츠 기반 라우팅(ALB)을 동시에 충족할 수 있다.
