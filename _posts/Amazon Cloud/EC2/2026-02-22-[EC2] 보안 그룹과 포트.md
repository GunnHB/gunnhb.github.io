## 보안 그룹
**보안 그룹**<sup>Security Group</sup>은 AWS의 가상 방화벽이다. 누가 외부에서 인스턴스로 들어올<sup>Inbound</sup> 수 있는지, 반대로 인스턴스에서 외부로 나갈<sup>Outbound</sup> 수 있는지를 통제한다.

### 특징
보안 그룹은 들어온 요청을 기억한다. 예를 들어 외부에서 인바운드 규칙을 통과해 들어온 트래픽은 아웃바운드 규칙에 상관없이 나갈 때 자동으로 통과된다. 반대로 사용자가 요청해서 나간 트래픽에 대한 응답도 무조건 들어올 수 있다.

> 기본적으로 인바운드 규칙은 모두 막혀있고 아웃바운드 규칙은 모두 허용되어 있다.
{: .prompt-tip}

추가로 **허용**<sup>Allow</sup> **규칙만이 존재**한다.

## 포트
**포트**<sup>port</sup>는 운영체제 통신에서 특정 프로세스나 네트워크 서비스를 식별하기 위한 **논리적인 접속점**<sup>Endpoint</sup>을 말한다. 서버에 도착한 패킷이 어떤 프로세스(프로그램)로 전달되어야 하는지를 식별하는 데 사용된다.

포트 번호는 0번부터 65,535번까지의 숫자로 할당된다. 그 중 0-1023번 구간을 잘 알려진 포트<sup>Well-known Ports</sup>라고 부르며, 이는 통신 규약에 의해 미리 목적이 지정되어 있다.

네트워크 환경에서 많이 사용되는 포트들은 다음과 같다.

- `22`
  - SSH 프로토콜
  - 보안 처리된 원격 터미널 접속

- `80`
  - HTTP 프로토콜
  - 암호화되지 않은 평문 웹 트래픽

- `443`
  - HTTPS 프로토콜
  - TLS/SSL로 암호화된 웹 트래픽

- `3306`
  - MySQL/MariaDB 통신 프로토콜
  - 데이터베이스 연결
