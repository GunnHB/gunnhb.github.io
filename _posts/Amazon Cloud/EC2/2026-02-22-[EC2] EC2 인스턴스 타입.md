## EC2 인스턴스 타입
EC2 인스턴스 타입<sup>Instance Type</sup>은 가상 서버의 하드웨어 스펙을 결정하는 모델명이다. 효율적인 아키텍쳐를 설계하기 위해서는 필수적으로 알아야 한다.

## EC2 인스턴스 타입의 구성
인스턴스 타입 이름은 다음과 같은 형태로 이루어져있다.

> [패밀리][세대][추가 기능].[크기]

### 패밀리
**패밀리**는 서버의 하드웨어 특성을 나타낸다.

- 범용 (M, T 시리즈)
  - CPU, 메모리, 네트워크가 밸런스 있게 맞춰진 모델
  - T 시리즈
    - 평소에는 CPU를 적게 쓰다가 트래픽이 갑자기 몰릴 때 CPU 성능을 버스트할 수 있음
    - 테스트 환경이나 트래픽이 적은 개인 블로그, 소규모 웹 서버에 적합
  - M 시리즈
    - T 시리즈보다 안정적으로 일정한 성능을 내는 표준 서버
    - 일반적인 엔터프라이즈 웹 서버에 많이 사용

- 컴퓨팅 최적화 (C 시리즈)
  - 메모리에 비해 CPU 비율이 높은 모델
  - 복잡한 연산에 적합
  - 게임 서버의 물리 연산 등

- 메모리 최적화 (R, X, Z 시리즈)
  - CPU에 비해 메모리가 넉넉한 모델
  - 데이터를 메모리에 올려두고 빠르게 처리해야한느 데이터베이스나 인메모리 캐시 서버에 적합

- 스토리지 최적화 (I, D, H 시리즈)
  - 로컬 디스크의 I/O 속도가 빠른 모델
  - 빅데이터 분석이나 대규모 로그 처리 등의 작업에 적합한 모델

- 가속화 컴퓨팅 (P, G, F 시리즈)
  - 고사양 GPU가 탑재된 모델
  - 머신러닝이나 고사양 3D 렌더링에 적합

### 세대
**세대**는 하드웨어가 얼마나 최신형인지를 나타낸다. 숫자가 높을수록 최신 세대이다.

### 추가 기능
**추가 기능**은 특정 하드웨어가 장착되었음을 나타낸다.

- `a`: AMD 프로세서 탑재
- `g`: AWS가 자체 개발한 프로세서 탑재 (Graviton)
- `i`: intel 프로세서 탑재
- `d`: 인스턴스에 직접 붙어있는 빠르고 임시적인 디스크를 포함

### 크기
**크기**는 가상 CPU의 개수와 메모리 용량을 나타낸다.

`nano` < `micro` < `small` < `large` < `xlarge` < `2xlarge` < `4xlarge`

사이즈는 한 단계 커질 때마다 CPU와 메모리, 가격이 2배씩 늘어난다.
