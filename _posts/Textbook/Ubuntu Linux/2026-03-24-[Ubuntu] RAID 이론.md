## RAID
- **RAID**<sup>Redundant Arrray of Inexpensive/Independent Disks</sup>
  - 여러 개의 디스크를 하나의 디스크처럼 사용하는 방식
  - 비용 절감
  - 높은 신뢰성
  - 성능 향상
  - 하드웨어 RAID
    - 하드웨어 제조 업체에서 여러 개의 디스크를 연결한 장비를 제작해 공급
    - 성능은 안정적이지만 가격이 높음
  - 소프트웨어 RAID
    - 운영체제에서 지원하는 방식
    - 신뢰성이 떨어지지만 가격이 저렴

## RAID 레벨
구성 방식에 따라 `Linear RAID`, `RAID 0`, `RAID 1`, `RAID 2`, `RAID 3`, `RAID 4`, `RAID 5`로 분류되며 실무에서는 `Linear RAID`, `RAID 0`, `RAID 1`, `RAID 5`, `RAID 6`, `RAID 1+0(RAID 10)`이 사용된다.

### 단순 볼륨
- 디스크 하나를 볼륨<sup>volume</sup>(묶음) 하나로 사용하는 방법
- RAID 방식에 **미포함**

### Linear RAID
- 두 개 이상의 디스크를 하나의 볼륨으로 사용
- 앞 디스크에 데이터가 완전히 저장된 후 다음 디스크에 데이터를 저장
- 앞 디스크에 데이터가 다 차지 않으면 다음 디스크는 전혀 사용되지 않는다.
- 100%의 공간 효율성

### RAID 0
- 두 개 이상의 디스크를 하나의 볼륨으로 사용
- **스트라이핑**<sup>striping</sup> 방식을 사용하여 데이터를 여러 디스크에 분산하여 동시 저장
- 장점
  - 빠른 I/O 속도
  - 100%의 공간 효율성
    - 단, 크기가 다른 장치를 묶었을 때 적은 크기만큼만 저장할 수 있음
- 단점
  - 장치 고장 시 데이터 복구 불가
- 빠른 성능을 요구하지만 유실에 치명적이지 않은 자료를 저장하는 데 적합
  - 캐시 데이터, 임시 데이터 저장 등

### RAID 1
- **미러링**<sup>morroring</sup> 방식을 사용해 두 개 이상의 디스크에 동일한 데이터를 중복 저장하는 방식
- 장점
  - 높은 안정성
    - 장치 중 하나가 고장나더라도 데이터는 안전
    - **결함 허용**<sup>fault-rolerance</sup>을 제공
- 단점
  - 50%의 공간 효율성
    - 같은 데이터를 다른 디스크에 저장하므로 실질적으로는 하나의 디스크만을 사용하는 것
- 비용이 많이 들더라도 유실되어서는 안되는 자료를 저장하는 데 적합
  - 회원 정보 데이터베이스, OS 및 핵심 구동 파일이 설치된 시스템 드라이브 등

### RAID 5
- 세 개 이상의 디스크를 하나의 볼륨으로 사용
- 스트라이핑과 **분산 페리티**<sup>distributed parity</sup> 방식을 결합하여 데이터를 여러 디스크에 분산 저장
  - 데이터 기록 시 오류 복구를 위한 검증 데이터인 **패리티**를 함께 생성
  - 모든 디스크에 번갈아 가며 고르게 분산 저장
  - 이를 통해 손실된 데이터를 유추 가능
- 장점
  - 장치 1대에 대한 결함 허용
    - 2대 이상 고장나면 복구 불가
  - 높은 공간 효율성
    - 전체 디스크 중 1개 분량의 용량만 패리티 저장에 사용
    - 디스크를 많이 사용할수록 공간 효율이 높아짐
- 단점
  - 리빌딩 위험성

### RAID 6
- 네 개 이상의 디스크를 하나의 볼륨으로 사용
- RAID 5의 개선
- 스트라이핑과 **이중 분산 패리티**<sup>double distributed parity</sup> 방식을 결합하여 데이터를 여러 디스크에 분산 저장
  - 서로 다른 수학적 알고리즘을 사용하여 2개의 패리티 데이터를 생성
  - 모든 디스크에 번갈아 가며 고르게 분산 저장
  - 이를 통해 손실된 데이터를 유추 가능
- 장점
  - 장치 2대에 대한 결함 허용
- 단점
  - RAID 5보다 느린 속도
  - RAID 5보다 낮은 공간 효율성
    - RAID 5: 장치 N대에 대한 N-1의 공간 효율설
    - RAID 6: 장치 N대에 대한 N-2의 공간 효율성

### RAID 1+0
- 네 개 이상의 디스크를 사용
- RAID 1<sup>morroring</sup>로 구성한 디스크 그룹을 RAID 0<sup>striping</sup>으로 구성하는 방식
  - 디스크를 두 개씩 짝지어 미러링 그룹으로 묶음 (RAID 1)
  - 미러링 그룹에 데이터를 분산 저장
- 장점
  - 빠른 I/O 속도
  - 높은 안정성
- 단점
  - 50%의 공간 효율성
