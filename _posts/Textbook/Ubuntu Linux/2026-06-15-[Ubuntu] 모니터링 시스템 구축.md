## 모니터링 시스템
서비스를 운영할 때 언제나 아무 문제없이 돌아간다면 좋겠지만 그럴 일은 극히 드물다. 만약 잘 돌아가던 서비스에 어느 순간 문제가 생겼다면 그 문제를 빠르게 파악하고 해결해야한다. 모니터링 시스템은 문제가 생기는 시점을 빠르게 확인할 수 있고, 스케일 업/아웃의 객관적인 지표가 된다.

### Prometheus + Grafana
요즘에는 `Datadog`나 `New Relic` 등 많은 모니터링 툴이 존재한다. 하지만 대부분의 현업에서 `Prometheus`와 `Grafana` 조합을 선택하는데, 둘의 장단점은 다음과 같다.

- Prometheus
  - 장점
    - 시계열 데이터 수집에 최적화
    - 서버에서 데이터를 끌어오는 방식이기 때문에 서버에 무리 없음
  - 단점
    - 직관적이지 않은 UI

- Grafana
  - 장점
    - 직관적인 UI
  - 단점
    - 데이터 저장 기능 없음

그렇기에 내 환경에도 두 조합을 이용해 모니터링 시스템을 구축하고자한다.

## 모니터링 시스템 구축

### 작업장 구축

1. 모니터링 전용 디렉토리 생성 및 이동

    ```bash
    # 모니터링 시스템을 위한 독립적 공간을 만들기 위해 홈으로 이동
    cd ~
    # 디렉토리 생성 및 이동
    mkdir monitoring && cd monitoring
    ```

2. Prometheus 지시서 생성

    ```bash
    vi prometheus.yml
    ```

    ```bash
    global:
    # 15초마다 상태를 측정
    scrape_interval: 15s 

    crape_configs:
      - job_name: 'prometheus'
        static_configs:
          # 일단 자기 자신의 상태부터 측정
          - targets: ['localhost:9090']
    ```

3. 두 툴을 띄울 도커 생성

    ```bash
    vi docker-compose.yml
    ```

    ```bash
    services:
      prometheus:
        image: prom/prometheus:latest
        container_name: prometheus
        ports:
          - "9090:9090"
        volumes:
          - ./prometheus.yml:/etc/prometheus/prometheus.yml

      grafana:
        image: grafana/grafana:latest
        container_name: grafana
        ports:
          - "3000:3000"
        environment:
          - GF_SECURITY_ADMIN_USER=admin
          - GF_SECURITY_ADMIN_PASSWORD=admin
    ```

4. 도커 실행

    ```bash
    docker compose up -d
    ```

### Node Exporter 연결
현재까지의 작업을 통해 프로메테우스와 그라파나의 연결을 확인했다. 그렇지만 프로메테우스에서는 가상머신의 상태를 측정하고 있지 않기 때문에 그라파나에서 대시보드를 만들 수 없는 상태이다. 이를 위해 `Node Exporter`라는 센서를 달아 서버의 상태를 읽어오도록 한다.

1. `prometheus.yml` 수정

    ```bash
    # ..
    # 새롭게 추가
    - job_name: 'ubuntu-server'
    static_configs:
      - targets: ['node-exporter:9100']
    ```

2. `docker-compose.yml` 수정

    ```bash
    # ..
    # 새롭게 추가
    node-exporter:
    image: prom/node-exporter:latest
    container_name: node-exporter
    ports:
      - "9100:9100"
    ```

이후 도커를 재실행해주면 `node-exporter`에 대한 도커를 새롭게 시작한다.

### 부하 테스트
마지막으로 `stress`를 통해 일정 시간동안 서버에 부하를 주도록 한다.

우선 즉각적인 변화를 확인하기 위해 그라파나의 업데이트 주기를 다음과 같이 변경한다.

![](/assets/img/post/Textbook/Ubuntu%20Linux/ubuntu_monitoring_12.png)
_최근 5분 기준, 5초마다 업데이트_

그런 다음 부하 테스트용 프로그램을 설치한다.

```bash
sudo apt-get update
sudo apt-get install -y stress
```

그리고 cpu에 60초 동안 부하를 준다.

```bash
# 가상의 일꾼 넷을 만들어 60초 동안 CPU를 100%로 사용하도록
stress --cpu 4 --timeout 60
```
