## 시작하기 전에
VMWare에 설치된 우분투에 하드디스크를 추가하려고 한다. 리눅스 환경에서는 하드디스크를 추가한 뒤 **논리적인 구조를 만들고 이를 시스템에 연결시켜야 한다.** 해당 포스트에서는 이에 대한 과정을 정리하려고 한다.

## 가상 하드디스크 추가
우선 하드디스크를 추가한다. 테스트용으로 작은 크기의 하드디스크를 추가했다.

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_1.png)
_1GB 크기의 하드디스크_

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_2.png){: width="650"}
_추가된 하드디스크_

> 참고로 VMWare에서는 4개의 SCSI 슬롯을 제공하며 각 SCSI 슬롯에는 15개의 디스크를 추가할 수 있다. 기존 하드디스크는 `SCSI 0:0`에 연결됐으며, 추가적인 하드디스크는 `0:1`, `0:2`, ... 에 연결된다.
{: .prompt-tip}

그 후 부팅하게 되면 2개의 하드디스크가 장착된 것을 확인할 수 있다.

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_3.png)
_왼쪽에 2개의 하드디스크_

## 파티션 할당
새로 장착한 하드디스크에 파티션을 할당한다. 리눅스 환경에서 하드디스크를 추가해 사용하려면 하나 이상의 파티션으로 나눠야 한다.

다음 명령어를 통해 파티션을 나눌 수 있다.

```bash
fdisk /dev/sdb          # SCSI 0:1 디스크 선택
Command : n             # 새로운 파티션 분할
Select : p              # Primary 파티션 선택
Partition number : 1    # 파티션 번호 1번 선택
First sector :          # 시작 섹터 번호 입력 (1개의 파티션을 계획했으므로 엔터 처리)
Last sector :           # 마지막 섹터 번호 입력 (1개의 파티션을 계획했으므로 엔터 처리)
Command : p             # 설정된 내용 확인
Command : w             # 설정 저장
```

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_4.png)
_명령어 입력_

## 포맷
다음으로 디스크를 포맷한다. 이는 파일 시스템을 `ext4` 형식으로 생성하는 과정으로, 명령어를 통해 진행된다.

```bash
mkfs.ext4 /dev/sdb1
```

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_5.png)
_명령어 입력_

## 마운트
`/dev/sdb1` 파일 시스템을 사용하기 위해 디렉토리에 마운트한다.

먼저 마운트할 `/mydata` 디렉토리를 만든다.

```bash
mkdir /mydata
```

그런 다음 해당 디렉토리에 /boot 디렉토리에 있는 파일 중 하나를 복사한다.

```bash
cp /boot/vmlinuz-6.8.0-31-generic /mydata/test1
```

이렇게 되면 `test1` 파일은 `/dev/sda2`에 위치한다.

다음으로 `/dev/sdb1` 장치를 `/mydata` 디렉토리에 마운트한다.

```bash
mount /dev/sdb1 /mydata
```

그런 다음 `ls -l /mydata`로 디렉토리 안을 확인하면 파일이 사라진 것처럼 보인다.

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_6.png)
_파일이 보이지 않는다_

이는 `/mydata` 디렉토리가 `/dev/sdb1`에 마운트되었기 때문에 파일이 `/dev/sda2`에 숨은 것이다. 이번에는 파일을 `test2`로 복사한 다음 다시 파일 목록을 확인하자.

```bash
cp /boot/vmlinuz-6.8.0-31-generic /mydata/test2
ls -l /mydata
```

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_7.png)
_새롭게 복사한 파일이 나타난다_

마운트된 상태에서 추가되는 파일들은 대상 장치에 저장된다. 그렇기에 새로 복사한 파일은 정상적으로 보이게 된다.

이번에는 마운트를 해제한 다음 파일 목록을 확인하자.

```bash
umount /dev/sdb1
ls -l /mydata
```

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_8.png)
_정상적으로 보이는 test1_

마운트를 해제하면 `test1` 파일이 정상적으로 나타난다. `test2` 파일은 삭제된 것이 아닌 `/dev/sdb1`에 위치하기 때문에 보이지 않는다. 필요하다면 아무 디렉토리에 `/dev/sdb1`을 마운트하면 test2를 언제든 사용할 수 있다.

## 마운트 유지
이번에는 재부팅 후에도 `/mydata`에 마운트가 유지되도록 한다.

`/etc/fstab` 파일을 열어 제일 아래에 다음을 추가한다.

```bash
/dev/sdb1 /mydata ext4  defaults  0 0
```

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_9.png)
_수정된 /etc/fstab 파일_

> `/etc/fstab` 파일은 리눅스가 부팅될 때마다 자동으로 읽는 중요한 파일이다. 이 파일에는 마운트 정보가 수록되어 있으며 글자가 틀릴 경우 아예 부팅이 되지 않을 수 있으니 주의해야 한다.
{: .prompt-tip}

파일을 저장한 뒤 재부팅한다. 그런 다음 다시 `/mydata`를 확인하면 부팅 시 자동으로 `/dev/sdb2`를 마운트하기 때문에 `test2` 파일을 확인할 수 있다.

![](/assets/img/post/Textbook/Ubuntu%20Linux/add_harddisk_10.png)
_재부팅 후에도 확인할 수 있는 test2 파일_
