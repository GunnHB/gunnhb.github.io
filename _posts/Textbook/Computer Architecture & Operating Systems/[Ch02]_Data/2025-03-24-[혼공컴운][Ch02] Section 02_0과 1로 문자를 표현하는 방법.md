## 개요
이전 절에서 언급한 것처럼 컴퓨터는 0과 1만을 이해하지 못한다. 그런데 우리가 입력한 문자를 어떻게 이해하고 모니터에 출력하는 걸까? 이번 절에서는 0과 1로 문자를 표현하는 방법, 즉 컴퓨터가 문자를 이해하고 표현하는 다양한 방법에 대해 알아보자.

## 문자 집합과 인코딩
0과 1로 문자를 표현하는 방법에 대해 알아보기 전에 반드시 알아야 할 세 가지 용어가 있다.

컴퓨터가 인식하고 표현할 수 있는 문자의 모음을 **문자 집합character set**이라고 한다. 컴퓨터는 문자 집합에 포함된 문자를 이해하지만 반대로 포함되지 않은 문자는 이해하지 못한다. 예를 들어 문자 집합이 \{a, b, c\}인 경우 d, e, f 등은 이해하지 못하는 것이다.

문자 집합에 포함된 문자라고 해서 컴퓨터가 그대로 이해할 수 있는 것은 아니다. 문자를 0과 1로 변환해야 비로소 이해할 수 있는데, 이 변환 과정을 **문자 인코딩character encoding**이라고 한다

> 인코딩 후 0과 1로 이루어진 결과값이 문자 코드가 된다.
{: .prompt-info}

반대로 0과 1로 이루어진 문자 코드를 사람이 이해할 수 있는 문자로 변환하는 과정을 **문자 디코딩character decoding**이라고 한다.

## 아스키 코드
**아스키ASCII**는 초창기 문자 집합 중 하나로, 알파벳, 아라비아 숫자, 일부 특수 문자 및 제어 문자를 포함한다. 아스키 문자들은 각각 7비트로 표현되는데, 7비트로 표현할 수 있는 정보의 가짓수는 2<sup>7</sup>개로, 총 128개의 문자를 표현할 수 있다.

> 실제로는 하나의 아스키 문자를 나타내기 위해 8비트(1바이트)를 사용한다. 하지만 8비트 중 1비트는 **패리티 비트parity bit**라고 불리는 ***오류 검출을 위해 사용되는 비트***이기 때문에 실질적으로 문자 표현을 위해 사용되는 비트는 7비트이다.
{: .prompt-info}

![](/assets/img/post/Textbook/ComputerArchitecture&OperatingSystems/ASCII.png)

위의 표는 아스키 코드표로, 문자들은 0부터 128개의 숫자 중 하나의 고유한 수에 일대일로 대응된다. 아스키 문자에 대응된 고유한 수를 아스키 코드라고 하며, 아스키 코드를 이진수로 표현함으로써 아스키 문자를 0과 1로 표현할 수 있다.

> 문자 인코딩에서 ‘글자에 부연되 고유한 값’을 코드 포인트code point라고 한다. 가령 A의 코드 포인트는 65이다.
{: .prompt-info}

아스키 코드는 매우 간단하게 인코딩된다는 장점이 있지만 한글을 표현하지 못한다는 치명적인 단점이 있다. 다시 말해 ***아스키 문자 집합 외의 다른 문자나 특수 문자는 표현하지 못한다.*** 이는 7비트로 문자를 표현하기에 128개보다 많은 문자를 표현하지 못하기 때문이다.

이를 보완하기 위해 각 나라의 언어를 인코딩할 수 있는 문자 집합과 인코딩 방식이 필요했다. 여기서 등장한 한글 인코딩 방식이 EUC-KR이다.

## EUC-KR
영어는 단어를 쓸 때 알파벳을 쭉 이어쓰면 되지만 한글을 각 음절 하나하나가 초성, 중성, 종성의 조합으로 이루어져 있다. 그래서 한글 인코딩에서는 **완성형 인코딩**과 **조합형 인코딩** 두 가지 방식이 존재한다.

- 완성형 인코딩
    - 초성, 중성, 종성의 조합으로 이루어진 완성된 하나의 글자에 고유한 코드를 부여하는 방식
- 조합형 인코딩
    - 초성을 위한 비트열, 중성을 위한 비트열, 종성을 위한 비트열을 할당하여 그것들의 조합으로 하나의 글자 코드를 완성하는 인코딩 방식

![](/assets/img/post/Textbook/ComputerArchitecture&OperatingSystems/EUC-KR_1.png)

***EUC-KR은 완성형 인코딩 방식***이다. 즉, EUC-KR 인코딩은 초성, 중성, 종성이 모두 결합된 한글 단어에 ***2바이트 크기의 코드를 부여***한다. 다시 말해 글자 하나를 표현하려면 16비트가 필요한 것이고, 이는 네 자리의 십육진수로 나타낼 수 있다.

![](/assets/img/post/Textbook/ComputerArchitecture&OperatingSystems/EUC-KR_2.png)
_EUC-KR로 인코딩된 한글 일부. ‘가’는 b0a0행의 두 번째 열인 b0a1<sub>(16)</sub>으로 인코딩 되는 것이다._

EUC-KR 방식은 총 2,350개 정도의 한글 단어를 표현할 수 있다. 하지만 이는 모든 한글을 표현할 수 있을 정도는 아니기 때문에 ‘쀍’, ‘쀓’, ‘믜’ 같은 글자는 표현할 수 없다.

## 유니코드와 UTF-8
EUC-KR 덕분에 한글을 표현할 수 있지만 모든 한글을 표현할 수 있는 것은 아니다. 더욱이 언어별 인코딩이 모두 다르다면 개발 시 굉장히 번거로울 것이다. 그래서 이를 보완하고자 등장한 것이 **유니코드unicode**이다.

유니코드는 EUC-KR보다 훨씬 다양한 한글을 포함하며 ***대부분 나라의 문자, 특수문자, 화살표나 이모티콘까지도 코드로 표현할 수 있는 통일된 문자 집합***이다. 추가로 유니코드의 인코딩 방식에는 UTF-8, UTF-16, UTF-32 등이 있다. 요컨데, UTF-8, UTF-16, UTF-32는 유니코드 문자에 부여된 값을 인코딩하는 방식이다.

> UTF는 Unicode Transformation Format의 약어로, 유니코드를 인코딩하는 방법이다.
{: .prompt-info}

UTF-8은 ***가변 길이 인코딩 방식***으로 1바이트부터 4바이트까지의 인코딩 결과를 만들어 낸다. UTF-8로 인코딩한 결과가 몇 바이트가 될지는 유니코드에 부여된 값(코드 포인트)의 범위에 따라 결정된다.

| 첫 코드 포인트 | 마지막 코드 포인트 | 1바이트 | 2바이트 | 3바이트 | 4바이트 |
| --- | --- | --- | --- | --- | --- |
| 0000 | 007F | 0XXXXXXX |  |  |  |
| 0080 | 07FF | 110XXXXX | 10XXXXXX |  |  |
| 0800 | FFFF | 1110XXXX | 10XXXXXX | 10XXXXXX |  |
| 10000 | 10FFF | 11110XXX | 10XXXXXX | 10XXXXXX | 10XXXXXX |

- 코드 포인트의 범위가 0부터 007F<sub>(16)</sub>까지는 1바이트로 표현
- 코드 포인트의 범위가 0080<sub>(16)</sub>부터 07FF<sub>(16)</sub>까지는 2바이트로 표현
- 코드 포인트의 범위가 0800<sub>(16)</sub>부터 FFFF<sub>(16)</sub>까지는 3바이트로 표현
- 코드 포인트의 범위가 10000<sub>(16)</sub>부터 10FFFF<sub>(16)</sub>까지는 4바이트로 표현

## 마무리

- **문자 집합**은 컴퓨터가 인식할 수 있는 문자의 모음으로 문자 집합에 속한 문자를 인코딩하여 0과 1로 표현할 수 있다.
- 아스키 문자 집합에 0부터 127까지의 수가 할당되어 **아스키 코드**로 인코딩된다.
- **EUC-KR**은 한글 2바이트 크기로 인코딩할 수 있는 완성형 인코딩 방식이다.
- **유니코드**는 여러 나라의 문자들을 광범위하게 표현할 수 있는 통일된 문자 집합이며, UTF-8, UTF-16, UTF-32는 유니코드 문자의 인코딩 방식이다.
