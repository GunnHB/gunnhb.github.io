## 개요
우리가 알아야 할 컴퓨터 구조 지식은 크게 두 가지이다.

- 컴퓨터가 이해하는 정보
- 컴퓨터의 네 가지 핵심 부품

이번 절에서는 지도를 그리 듯 컴퓨터 구조의 큰 그림을 그려나갈 것이다. 그렇기에 낯선 용어가 많이 등장하지만 눈도장 정도만 찍어두도록 하자.

## 컴퓨터가 이해하는 정보
**컴퓨터는 0과 1로 표현된 정보만을 이해**한다. 그리고 이렇게 0과 1로 표현되는 정보에는 크게 두 종류가 있는데, **데이터**와 **명령어**이다.

- 컴퓨터가 이해하는 정보
    - 데이터
    - 명령어

**데이터**란 컴퓨터가 이해하는 숫자, 문자, 이미지, 동영상과 같은 **정적인 정보**이다. 이는 컴퓨터와 주고받는 정보나 컴퓨터에 저장된 정보를 통칭하기도 한다.

> 데이터란 곧 0과 1로 이루어진 숫자나 문자 등을 의미한다.
{: .prompt-info}

**명령어**란 컴퓨터를 실질적으로 동작시키는 정보이다. 데이터는 명령어를 위한 일종의 재료로, 명령어를 통해 데이터가 움직이고 컴퓨터가 작동하게 된다.

> 프로그램은 ‘명령어들의 모음’으로 정의되기도 한다.
{: .prompt-info}

## 컴퓨터의 네 가지 핵심 부품

세상에는 다양한 종류의 컴퓨터가 존재한다. 외관과 형태를 막론하고 컴퓨터를 이루는 핵심 부품은 크게 다르지 않다.

- 컴퓨터의 네 가지 핵심 부품
    - CPU
    - 메모리
    - 보조기억장치
    - 입출력장치

### 메모리
**메모리**는 **현재 실행되는 프로그램**의 명령어와 데이터를 저장하는 부품이다. 즉, 프로그램이 실행되려면 반드시 메모리에 저장되어 있어야 한다.

메모리에 저장할 수 있는 명령어와 데이터는 굉장히 많다. 이 많은 정보들 중 사용자가 원하는 데이터나 명령어에 접근할 수 있어야 하는데, 여기서 사용되는 개념이 주소address이다. 주소를 사용하면 원하는 정보에 빠르고 효율적으로 접근할 수 있게 된다.

메모리에서 기억해야 할 점은 다음과 같다.

- 프로그램이 실행되기 위해서는 반드시 메모리에 저장되어 있어야 한다.
- 메모리는 실행되는 프로그램의 명령어와 데이터를 저장한다.
- 메모리에 저장된 값의 위치는 주소로 알 수 있다.

### CPU
**CPU**는 컴퓨터의 두뇌이다. CPU는 메모리에 저장된 명령어를 읽어 들이고, 읽어 들인 명령어를 해석하고, 실행하는 부품이다.

CPU의 내부 구성 요소는 다음과 같다.

- 산술논리연산장치(ALU: Arithmetic Logic Unit)
    - 쉽게 말해 계산기
    - 컴퓨터 내부에서 수행되는 대부분의 계산을 수행
- 레지스터(Register)
    - CPU 내부의 작은 임시 저장 장치
    - 프로그램을 실행하는 데, 필요한 값들을 임시로 저장
    - CPU 내부에는 여러 개의 레지스터가 존재하고, 각기 다른 이름과 역할을 가짐
- 제어장치(CU: Control Unit)
    - 제어 신호를 내보내고 명령어를 해석하는 장치
        - **제어 신호**란 컴퓨터 부품들을 관리하고 작동시키기 위한 일종의 전기 신호
        - CPU가 메모리에 저장된 값을 읽고 싶을 땐 메모리를 향해 **메모리 읽기**라는 제어 신호를 보낸다.
        - CPU가 메모리에 어떤 값을 저장하고 싶을 땐 메모리를 향해 **메모리 쓰기**라는 제어 신호를 보낸다.

CPU에서 기억해야 할 점은 다음과 같다.

- CPU는 메모리에 저장된 값을 읽어 들이고, 해석하고, 실행하는 장치이다.
- CPU내부에는 ALU, 레지스터, 제어장치가 있다.
    - ALU는 계산하는 장치
    - 레지스터는 임시 저장 장치
    - 제어장치는 제어 신호를 발생시키고 명령어를 해석하는 장치

### 보조기억장치
앞서 살펴본 메모리는 두 가지의 치명적이 단점이 있는데, 첫째는 용량에 비해 가격이 비싸다는 점이고 둘째는 전원이 꺼지면 저장된 내용을 잃는다는 점이다.

> 현재 실행되는 프로그램은 메모리에 저장되는데, 메모리는 전원이 꺼지면 저장된 내용이 날아가기 때문
{: .prompt-info}

이에 메모리보다 크기가 크고 전원이 꺼져도 저장된 내용을 잃지 않는 메모리가 필요하게 되었는데, 이것이 바로 **보조기억장치**이다. 우리가 어떤 프로그램을 설치하고 전원을 껐다 켜도 날아가지 않는 이유가 바로 ***보조기억장치에 저장되기 때문***이다. 즉, 메모리는 실행할 정보를 저장하고, 보조기억장치는 보관할 정보를 저장한다.

### 입출력장치
**입출력장치**는 마이크, 스피커, 프린터, 마우스, 키보드처럼 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 교환하는 장치를 의미한다.

그런데 여기서 하나 궁금증이 생길 수 있다. “외장하드나 USB도 컴퓨터 외부에 연결되어 정보를 교환하는데, 이것도 입출력장치인가?” 사실 여러 서적이나 문서에서는 이 두 장치를 나눠 설명하지만 딱 잘라 구분되는 개념은 아니다. 다만 보조기억장치는 일반적인 입출력장치에 비해 **메모리를 보조**하는 특별한 기능을 수행하는 입출력장치라고 이해할 수 있다.

### 메인보드
지금까지 설명한 컴퓨터의 핵심 부품들은 **메인보드main board**라는 판에 연결된다. 메인보드에 연결된 부품들은 서로 정보를 주고받을 수 있는데, 이는 메인보드 내부에 **버스bus**라는 통로를 통해 이뤄진다. 컴퓨터 내부에는 다양한 종류의 버스가 존재하는데, 이 여러 버스 가운데 컴퓨터의 네 가지 핵심 부품을 연결하는 버스가 **시스템 버스system bus**이다.

시스템 버스에는 **주소 버스address bus**, **데이터 버스data bus**, **제어 버스control bus**로 구성되어 있다.

- 주소 버스
    - 주소를 주고받는 통로
- 데이터 버스
    - 명령어와 데이터를 주고받는 통로
- 제어 버스
    - 제어 신호를 주고받는 통로

## 마무리
- 컴퓨터가 이해하는 정보에는 데이터와 명령어가 있다.
- 메모리는 현재 실행되는 프로그램의 명령어와 데이터를 저장하는 부품이다.
- CPU는 메모리에 저장된 명령어를 읽어 들이고, 해석하고, 실행하는 부품이다.
- 보조기억장치는 전원이 꺼져도 보관할 프로그램을 저장하는 부품이다.
- 입출력장치는 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 교환할 수 있는 부품이다.
- 시스템 버스는 컴퓨터의 네 가지 핵심 부품들이 서로 정보를 주고받는 통로이다.
