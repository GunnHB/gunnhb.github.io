## 라우터
- **라우팅**<sup>routing</sup>
  - 패킷이 이동할 최적의 경로를 설정한 뒤 해당 경로로 패킷을 이동시키는 것

- **라우터**<sup>router</sup>
  - 라우팅을 실행하는 네트워크 계층의 네트워크 장비

- **홉**<sup>hop</sup>
  - 라우터와 라우터 간에 이동하는 하나의 과정
  - 서로 다른 LAN에 존재하는 호스트 간의 통신 과정에서 패킷을 서로 도달하기까지 여러 라우터를 거친다.

## 라우팅 테이블
- **라우팅 테이블**<sup>routing table</sup>
  - 특정 수신지까지 도달하기 위한 정보를 명시한 정보
  - 라우터가 저장하고 관리

- 라우팅 테이블의 정보
  - **다음 수신지 IP 주소**와 **서브넷 마스크**
    - 최종적으로 패킷을 전달할 대상

  - **다음 홉**
    - 최종 수신지까지 가기 위해 다음으로 거쳐야 할 호스트의 IP 주소나 인터페이스
    - 게이트웨이라고도 명시된다.

  - **네트워크 인터페이스**
    - 패킷을 내보낼 통로
    - NIC가 직접적으로 명시되거나 대응되는 IP 주소가 명시되기도 한다.

  - **메트릭**
    - 해당 경로로 이동하는 데 드는 비용
    - 라우터는 메트릭이 낮은 경로를 선호한다.

라우팅 테이블에 없는 경로로 패킷을 전송해야 할 경우 **디폴트 라우트**<sup>default route</sup>라는 기본 경로로 보낸다. 이는 모든 IP 주소를 의미하는 `0.0.0.0/0`로 명시되며 패킷은 **기본 게이트웨이로** 전달된다.

## 정적 라우팅과 동적 라우팅
- **정적 라우팅**<sup>static routing</sup>
  - 사용자가 직접 채워 넣은 라우팅 테이블을 토대로 라우팅되는 방식
  - 명령어를 통해 추가할 수 있다.
  - 관리할 라우터가 늘어나면 관리가 힘들다.

- **동적 라우팅**<sup>dynamic routing</sup>
  - 자동으로 라우팅 테이블을 만들고 이를 토대로 라우팅되는 방식
  - 네트워크 경로상에 문제가 발생할 경우 우회할 수 있도록 경로를 자동으로 갱신한다.
  - 라우팅 프로토콜을 통해 테이블을 갱신한다.

동일한 라우팅 정책으로 운용되는 라우터들의 집단 네트워크를 **AS**<sup>Autonomous System</sup>라고 한다. AS 내의 라우터들은 내부에서만 통신할 수도, 외부와 통신할 수도 있다. 외부로 통신할 경우 AS 경계에서 AS 내외로 통신을 주고받을 수 있는 **AS 경계 라우터**<sup>ASBR: Autonomous System Boundary Router</sup>라는 특별한 라우터를 이용한다.

## 라우팅 프로토콜
- **라우팅 프로토콜**<sup>routing protocol</sup>
  - 라우터끼리 자신들의 정보를 교환하며 패킷이 이동할 최적의 경로를 찾기 위한 프로토콜
  - 종류는 두 가지
    - **IGP**<sup>Interior Gateway Protocol</sup>
      - AS 내부에서 수행
    - **EGP**<sup>Exterior Gateway Protocol</sup>
      - AS 외부에서 수행

### IGP: RIP와 OSPF
- **RIP**<sup>Routing Information Protocol</sup>
  - IGP의 라우팅 방식 중 하나
  - **거리 벡터**<sup>distance vector</sup> 기반의 라우팅 프로토콜
    - 거리를 기반으로 최적의 경로를 찾는 라우팅 프로토콜
    - 거리는 ***홉의 수***를 의미한다.

RIP는 인접한 라우터끼리 경로 정보를 주기적으로 주고받으며 라우팅 테이블을 갱신한다. 이를 통해 ***홉의 수가 가장 적은 경로***를 최적의 경로라고 판단한다. 홉의 수가 적을수록 라우팅 테이블의 메트릭 값도 작아진다.

- **OSPF**<sup>Open Shortest Path First</sup>
  - IGP의 라우팅 방식 중 하나
  - **링크 상태**<sup>link state</sup> 라우팅 프로토콜
    - 호스트(노드)들이 통신 매체(간선)로 연결된 즉, 그래프 형태로 저장한다.
    - 그래프 정보는 **링크 상태 데이터베이스**<sup>LSDB: Link State DataBase</sup>에 저장된다.

OSPF는 대역폭을 기반으로 메트릭을 계산한다. ***대역폭이 높은 링크***일수록 메트릭이 낮은 경로로 인식한다. 또한 네트워크 구성이 변경되었을 때 라우팅 테이블이 갱신된다.

추가로 네트워크 규모가 커졌을 때 OSPF 방식의 라우팅 태이블 갱신은 부담이 크다. 그렇기에 OSPF에서는 AS를 **에어리어**<sup>area</sup>라는 단위로 나누고 구분된 에어리어 내에서만 링크 상태를 공유한다. 에어리어에는 번호가 부여되며 에어리어 경계에 있는 **ABR**<sup>Area Border Router</sup>이라는 라우터가 에어리어 간의 연결을 담당한다.

### EGP: BGP
- **BGP**<sup>Border Gateway Protocol</sup>
  - AS 간의 통신에서 사용되는 대표적인 프로토콜
  - EGP 범주 내에 속한다.
  - AS 내·외부의 통신도 가능하다.
    - **eBGP**<sup>external BGP</sup>
      - AS 간의 통신
    - **iBGP**<sup>internal BGP</sup>
      - AS 내의 통신

AS 간에 정보를 주고받기 위해서는 AS 내에서 eBGP를 사용하는 라우터가 하나 이상 있어야 하고, 다른 AS의 eBGP 라우터와 연결되어야 한다. 이렇게 연결된 eBGP 라우터를 **피어**<sup>peer</sup>라고 하며, 피어 관계가 되도록 연결되는 과정을 **피어링**<sup>peering</sup>이라고 한다.

BGP는 수신지 주소와 더불어 '속성'과 '정책'이 고려되기 때문에 최적 경로를 결정하는 과정이 복잡하다.

- BGP의 속성
  - **AS-PATH 속성**
    - 메시지가 수신지에 이르는 과정을 통과하는 AS들의 목록
    - 메시지가 AS를 거칠 때마다 거쳐간 AS가 추가된다.
    - eBGP의 특징
      - AS 간 라우팅을 할 때 거치게 될 라우터의 수가 아닌 ***AS의 수를 고려***한다.
      - RIP처럼 단순히 수신지에 이르는 거리가 아닌 메시지가 ***어디를 거쳐 어디로 이동하는지를 나타내는 경로를 고려***한다.

  - **NEXT-HOP 속성**
    - 다음으로 거칠 라우터의 IP 주소
  
  - **LOCAL-PREF 속성**
    - AS 외부 경로에 있어 AS 내부에서 어떤 경로를 선호할지에 대한 척도
    - LOCAL PREFerence의 약자
    - 경로를 선택하는 과정에서 앞선 속성보다 우선시된다.
    - 값이 클수록 우선으로 선택
