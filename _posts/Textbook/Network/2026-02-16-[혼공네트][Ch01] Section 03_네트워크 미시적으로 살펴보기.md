## 프로토콜
**프로토콜**<sup>Protocol</sup>은 노드 간에 정보를 올바르게 주고받기 위해 합의된 규칙이나 방법이다. 서로 다른 통신 장치들이 정보를 주고받으려면 프로토콜이 통해야 한다.

## 네트워크 참조 모델
**네트워크 참조 모델**<sup>network reference model</sup>은 통신이 일어나는 각 과정을 계층으로 나눈 구조이다. 계층으로 표현한다는 점에서 **네트워크 계층 모델**이라 부르기도 한다.

통신 과정을 계층으로 나누면 계층의 목적에 맞는 프로토콜과 네트워크 장비를 구성할 수 있어 **네트워크 구성과 설계가 용이**하며 통신 과정에서 문제가 발생했을 때 문제 발생 지점을 추측할 수 있어 **문제 진단과 해결이 용이**하다. 

### OSI 모델
**OSI 모델**은 국제 표준화 기구<sup>OSI: International Organization for Standardization</sup>에서 만든 네트워크 참조 모델이다.

7계층으로 나뉘며 각각 물리 계층<sup>1계층</sup>, 데이터 링크 계층<sup>2계층</sup>, 네트워크 계층<sup>3계층</sup>, 전송 계층<sup>4계층</sup>, 세션 계층<sup>5계층</sup>, 표현 계층<sup>6계층</sup>, 응용 계층<sup>7계층</sup>이다.

#### 물리 계층
**물리 계층**<sup>physical layer</sup>은 1과 0으로 표현되는 비트 신호(비트 데이터)를 주고받는 계층이다. 통신 매체에 맞는 신호로 운반되도록 비트 데이터의 변환이 이루어지고 통신 매체를 통한 송수신이 이루어진다.

#### 데이터 링크 계층
**데이터 링크 계층**<sup>data link layer</sup>은 네트워크 내 주변 장치 간의 정보를 올바르게 주고받기 위한 계층이다. 물리 계층을 통해 주고받는 정보에 오류가 없는지 확인하고 MAC 주소 체계를 통해 네트워크 내 송수신지를 특정할 수 있다.

#### 네트워크 계층
**네트워크 계층**<sup>network layer</sup>은 메시지를 다른 네트워크에 속한 수신지까지 전달하는 계층이다. IP 주소 체계를 통해 통신하고자 하는 수신지 호스트와 네트워크를 식별하고 원하는 수신지에 도달하기 위한 최적의 경로를 결정한다.

#### 전송 계층
**전송 계층**<sup>transport layer</sup>은 신뢰성 있고 안정성 있는 전송을 해야할 때 필요한 계층이다. 패킷의 흐름을 제어하거나 전송 오류를 점검해 신뢰성 있고 안정적인 전송이 이루어지도록 한다.

#### 세션 계층
**세션 계층**<sup>session layer</sup>은 세션을 생성하거나 유지하고 종료되었을 때 끊어주는 역할을 담당한다.

> **세션**<sup>session</sup>은 통신을 주고받는 호스트의 응용 프로그램 간에 연결 상태를 의미한다.
{: .prompt-tip}

#### 표현 계층
**표현 계층**<sup>presentation layer</sup>은 사람의 문자를 코드로 변환하거나 압축, 암호화같은 작업이 이루어진다.

#### 응용 계층
**응용 계층**<sup>application layer</sup>은 사용자 및 사용자가 이용하는 응용 프로그램에 가장 밀접한 계층이다. 웹 브라우저 프로그램에 웹 페이지를 제공하거나 이메일 클라이언트 프로그램에 송수신된 이메일을 제공하는 등의 실질적인 네트워크 서비스가 제공된다.

### TCP/IP 모델
OSI 모델이 '이상적 설계'에 가깝다면 **TCP/IP 모델**은 '실용적 구현'에 가까운 참조 모델이다. 

4계층으로 나뉘며 각각 네트워크 액세스 계층<sup>1계층</sup>, 인터넷 계층<sup>2계층</sup>, 전송 계층<sup>3계층</sup>, 응용 계층<sup>4계층</sup>이다.

> TCP/IP 모델은 TCP/IP 4계층, 인터넷 프로토콜 스위트<sup>internet protocol suite</sup>, TCP/IP 프로토콜 스택<sup>protocol stack</sup>이라고도 불린다.
{: .prompt-tip}

#### 네트워크 액세스 계층
**네트워크 액세스 계층**<sup>network access layer</sup>은 **링크 계층**<sup>link layer</sup>, **네트워크 인터페이스 계층**<sup>network interface layer</sup>이라고도 불린다. OSI 모델의 데이터 링크 계층과 유사하다.

#### 인터넷 계층
**인터넷 계층**<sup>internet layer</sup>은 OSI 모델의 네트워크 계층과 유사하다.

#### 전송 계층
**전송 계층**<sup>transport layer</sup>은 OSI 모델의 전송 계층과 유사하다.

#### 응용 계층
**응용 계층**<sup>application layer</sup>은 OSI 모델의 세션 계층, 표현 계층, 응용 계층을 합친 것과 유사하다.

## 캡슐화와 역캡슐화
패킷은 송신 과정에서 **캡슐화**가 이루어지고 수신 과정에서 **역캡슐화**가 이루어진다.

### 캡슐화
어떤 정보를 송신할 때 각 계층에서는 상위 계층으로부터 내려받은 패킷을 페이로드로 삼아 프로토콜에 걸맞은 헤더(혹은 트레일러)를 덧붙인 후 하위 계층으로 전달한다. 이렇게 송신 과정에서 헤더 및 트레일러를 추가해 나가는 과정을 **캡슐화**<sup>encapsulation</sup>라 부른다.

### 역캡슐화
역캡슐화는 송신 과정의 반대이다. 캡슐화 과정에서 붙였던 헤더 및 트레일러를 각 계층에서 확인한 뒤 제거한다. 이를 **역캡슐화**<sup>decapsulation</sup>라 부른다.

## PDU
각 계층에서 송수신되는 메시지의 단위를 **PDU**<sup>Protocol Data Unit</sup>라고 한다. 즉 ***각 계층마다의 PDU가 존재***한다.

![](/assets/img/post/Textbook/Network/PDU.png)

> '패킷'이라는 용어는 패킷 교환 네트워크에서 쪼개어져 전송되는 단위를 통칭하기도 하지만 네트워크 계층에서의 송수신 단위를 지칭하기도 한다. 서적에서는 후자를 **IP 패킷**이라 칭한다.
{: .prompt-tip}

## 좀 더 알아보기
### OSI 7계층, TCP/IP 4계층은 사실 아무것도 해주지 않는다
네트워크 참조 모델은 모든 프로토콜과 네트워크 장비가 반드시 지켜야 하는 규칙이나 법규가 아닌 그저 가이드라인이다. ***프로토콜 혹은 네트워크 장비 같은 참조 모델이 속한 대상이 네트워크를 작동시키는 주체***인 것이지 참조 모델이나 특정 계층은 네트워크 구조에 대한 개념 참조에 사용하는 것이 바람직하다.

### 트래픽과 네트워크 성능 지표
**트래픽**<sup>traffic</sup>이란 네트워크 내의 정보량을 의미한다. 특정 노드에 트래픽이 몰린다는 것은 해당 노드가 특정 시간에 처리해야 할 정보가 많음을 의미하고 이는 해당 노드에 **과부하**<sup>overhead</sup>가 생길 수 있다.

네트워크 성능을 평가할 수 있는 지표에는 처리율, 대역폭, 패킷 손실이 있다.

#### 처리율
**처리율**<sup>throughput</sup>은 단위 시간당 네트워크를 통해 실제로 전송되는 정보량을 의미한다.

#### 대역폭
**대역폭**<sup>bandwidth</sup>은 단위 시간당 통신 매체를 통해 송수신할 수 있는 최대 정보량을 의미한다.

#### 패킷 손실
**패킷 손실**<sup>packet loss</sup>은 송수신되는 패킷이 손실되는 상황을 의미한다. 높은 트래픽으로 인해 노드가 순간적으로 처리해야 할 패킷이 너무 많아지거나 네트워크상에서 예기치 못한 장애가 발생해서 패킷을 처리하지 못할 경우 발생할 수 있다.

> 패킷 손실은 전체 패킷 중 유실된 패킷을 백분위로 표현한 값을 사용하는 경우가 많다.
{: .prompt-tip}
